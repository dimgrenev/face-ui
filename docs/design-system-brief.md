# Face UI React Design System Brief

## Goals
- Clean component taxonomy with explicit canonical names.
- First-class responsive surfaces instead of ad-hoc mobile overrides.
- Governance metadata that makes maturity, review status, and replacements visible.

## Success Criteria
- Preview parity between gallery cards and standalone component preview.
- Stable public APIs with compatibility shims for renamed components.
- Responsive overlays share one contract instead of per-component exceptions.
- Every public component has category, maturity, responsive behavior, and related-component guidance.
